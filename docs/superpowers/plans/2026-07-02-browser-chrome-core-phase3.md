# Browser Chrome Core Phase 3 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Expose the next stable Codex Chrome-compatible surface in Lume: tab cleanup, Chrome tab claiming, and conservative Playwright core methods.

**Architecture:** Keep the existing client/runtime split. Runtime descriptor `apiSupportOverrides` decides what agents can see, while `BrowserClient` supplies matching methods and the runtime dispatcher continues serving existing command implementations. No new dependencies.

**Tech Stack:** TypeScript, Chrome extension runtime, Node `node:test`, npm scripts in `plugins/lume-chrome`.

---
